locations

The following methods allow for interaction with the ZIA Location Management API endpoints.

Methods are accessible via zia.locations

class LocationsAPI
add_location(name, **kwargs)

Adds a new location.

Parameters:

name (str) – Location name.

Keyword Arguments:

  • ip_addresses (list) –

    For locations: IP addresses of the egress points that are provisioned in the Zscaler Cloud. Each entry is a single IP address (e.g., 238.10.33.9).

    For sub-locations: Egress, internal, or GRE tunnel IP addresses. Each entry is either a single IP address, CIDR (e.g., 10.10.33.0/24), or range (e.g., 10.10.33.1-10.10.33.10)).

  • ports (list of str) – List of whitelisted Proxy ports for the location.

  • vpn_credentials (dict) – VPN credentials for the location.

Returns:

The newly created location resource record

Return type:

Box

Examples

Add a new location with an IP address.

>>> zia.locations.add_location(name='new_location',
...    ip_addresses=['203.0.113.10'])

list_locations(**kwargs)

Returns a list of locations.

Keyword Arguments:

  • **auth_required (bool, optional) – Filter based on whether the Enforce Authentication setting is enabled or disabled for a location.

  • **bw_enforced (bool, optional) – Filter based on whether Bandwith Control is being enforced for a location.

  • **max_items (int, optional) – The maximum number of items to request before stopping iteration.

  • **max_pages (int, optional) – The maximum number of pages to request before stopping iteration.

  • **page_size (int, optional) – Specifies the page size. The default size is 100, but the maximum size is 1000.

  • **search (str, optional) – The search string used to partially match against a location’s name and port attributes.

  • **xff_enabled (bool, optional) – Filter based on whether the Enforce XFF Forwarding setting is enabled or disabled for a location.

Returns:

List of configured locations.

Return type:

BoxList

Examples

List locations using default settings:

>>> for location in zia.locations.list_locations():
...    print(location)

List locations, limiting to a maximum of 10 items:

>>> for location in zia.locations.list_locations(max_items=10):
...    print(location)

List locations, returning 200 items per page for a maximum of 2 pages:

>>> for location in zia.locations.list_locations(page_size=200, max_pages=2):
...    print(location)

list_sub_locations(location_id, **kwargs)

Returns sub-location information for the specified location ID.

Parameters:

  • location_id (str) – The unique identifier for the parent location.

  • **kwargs – Optional keyword args.

Keyword Arguments:

  • **auth_required (bool, optional) – Filter based on whether the Enforce Authentication setting is enabled or disabled for a location.

  • **bw_enforced (bool, optional) – Filter based on whether Bandwith Control is being enforced for a location.

  • **enable_firewall (bool, optional) – Filter based on whether Enable Firewall setting is enabled or disabled for a sub-location.

  • **enforce_aup (bool, optional) – Filter based on whether Enforce AUP setting is enabled or disabled for a sub-location.

  • **max_items (int, optional) – The maximum number of items to request before stopping iteration.

  • **max_pages (int, optional) – The maximum number of pages to request before stopping iteration.

  • **page_size (int, optional) – Specifies the page size. The default size is 100, but the maximum size is 1000.

  • **search (str, optional) – The search string used to partially match against a location’s name and port attributes.

  • **xff_enabled (bool, optional) – Filter based on whether the Enforce XFF Forwarding setting is enabled or disabled for a location.

Returns:

A list of sub-locations configured for the parent location.

Return type:

BoxList

Examples

>>> for sub_location in zia.locations.list_sub_locations('97456691'):
...    pprint(sub_location)
update_location(location_id, **kwargs)

Update the specified location.

Note: Changes are not additive and will replace existing values.

Parameters:

  • location_id (str) – The unique identifier for the location you are updating.

  • **kwargs – Optional keyword args.

Keyword Arguments:

  • ip_addresses (list of str) – List of updated ip addresses.

  • ports (list of str) – List of whitelisted Proxy ports for the location.

  • vpn_credentials (dict) – VPN credentials for the location.

Returns:

The updated resource record.

Return type:

Box

Examples

Update the name of a location:

>>> zia.locations.update('97456691',
...    name='updated_location_name')

Upodate the IP address of a location:

>>> zia.locations.update('97456691',
...    ip_addresses=['203.0.113.20'])
